The problem

Modern inference is just spinning up vLLM
a multi-axis optimization problem.

Every model has dozens of correct ways to serve it, and only one peak. Engine choice, batch size, KV-cache layout, tensor parallelism, quantization scheme, custom kernels — the search space is massive, and a wrong setting silently leaves 40–80% of your GPU on the floor.

The platform

Four agents. One autonomous pipeline.

Each stage is owned by a specialized agent. Together they take you from model weights to a hardened, peak-throughput deployment — without you writing a single config line.

01

Engine selection

Profiles your workload pattern — chat, RAG, batch, agentic, vision-language — and ranks vLLM, SGLang, TensorRT-LLM, TokenSpeed, and others against your exact distribution.

  • Workload fingerprinting
  • Engine capability matrix
  • Cross-engine A/B harness
02

Config tuning

Bayesian + multi-armed bandit search over batch size, max-seqs, KV-cache paging, TP/PP shape, CUDA graphs, speculative decoding, and quantization (AWQ / GPTQ / FP8 / INT4).

  • 500+ hyperparameters explored
  • SLO-aware Pareto front
  • Closed-loop re-tuning on drift
03

Kernel synthesis

Writes custom Triton and CUDA kernels for the hot path of your model — fused RMSNorm+RoPE, paged FlashAttention variants, MoE routing, prefix-shared prefill. Auto-verified bit-equivalent to reference.

  • Triton / CUDA / ROCm / Metal targets
  • 0-ulp output verification
  • Per-arch tuning (SM90, SM100, CDNA3)
04

Validation

Every deployment ships behind a generated test harness: functional correctness, golden-output regression, k6 load tests at projected QPS, and adversarial security / prompt-injection battery.

  • Correctness vs reference model
  • Load tests up to 10k concurrent
  • OWASP LLM Top-10 coverage
How it works

Three inputs. One deployment.

You bring the model and the hardware. We bring the rest.

  1. 1

    Pick a model

    Hugging Face checkpoint, local weights, custom fine-tune, or one of 600+ pre-profiled models from our catalog.

    python ⎘ copy 
    from autoinference import Deployment

deploy = Deployment(
    model="meta-llama/Llama-3.1-70B-Instruct",
    quantize="auto",   # picks AWQ / FP8 / INT4 for your hw
)
  2. 2

    State your hardware

    A single H100, a multi-node H200 cluster, AMD MI300X, edge Jetson, or a cloud provider. We auto-detect topology, interconnect, memory, and headroom.

    python ⎘ copy 
    deploy.target(
    hardware="8x H100 SXM",
    interconnect="NVLink",
    region="us-west-2",
    slo={"p99_latency_ms": 200, "min_qps": 120},
)
  3. 3

    Hit deploy

    The agentic harness takes over: profile → choose engine → tune configs → synthesize kernels → load-test → harden → ship. You get an OpenAI-compatible endpoint, with full audit trail.

    python ⎘ copy 
    endpoint = deploy.run()
# → https://api.autoinference.org/v1/llama-3.1-70b
# → tested, hardened, optimized for your exact workload
